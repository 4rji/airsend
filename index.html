Fast file transfer and chat using room codes

AirSend is a lightweight Go project that lets users share files and chat through a web interface or CLI. It combines a simple browser UI with a QUIC-based transport layer, so teams can send data quickly and keep communication in one place.

What the project is about

Flexible by design

AirSend was built to avoid rigid defaults and allows custom hosts, ports, and deployment styles.

Dual experience

Users can work from a modern web page or from terminal commands, depending on their workflow.

Room-based sharing

A code links people to the same transfer or chat room, making collaboration direct and simple.

How it works

Step 01

Start AirSend with web + QUIC mode. The web UI runs on a TCP port and data transport runs on a UDP port.

Step 02

Upload a file in the web UI or send from CLI. A room code is generated (or provided) for access.

Step 03

Receivers enter the same code to download files or join the shared chat room from web or terminal.

Step 04

Messages are broadcast to all users in that room, and connection events can be logged for visibility.

sudo go run . -sw 0.0.0.0 3888 0.0.0.0 443
airsend -f file.pdf
airsend -r room42
airsend -m room42

Main functionalities

  • Web upload and forced-download flow to avoid inline preview issues.
  • Multi-user chat rooms over WebSocket with Enter-to-send support.
  • CLI and web interoperability using the same room code model.
  • QUIC keepalive and idle timeout handling for connection stability.
  • Configurable host/port settings for local, remote, or custom deployments.
  • Connection logging to track web and QUIC activity.

Built for practical sharing: quick setup, direct transfer, and real-time communication.
